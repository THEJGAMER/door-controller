# DoorControl Pro 🛡️

An enterprise-grade, real-time web management interface for UHPPOTE TCP/IP Wiegand Access Controllers. Built with Node.js, SQLite, and WebSockets for high-performance multi-site security management.

## 🚀 Key Features

### 🔐 Hardened OIDC Security
- **Back-Channel Logout**: Support for IDP-initiated logout (e.g., via Keycloak) to instantly invalidate local sessions.
- **Active Session Validation**: Background heartbeat checks Keycloak every 60 seconds to ensure user sessions haven't been revoked or disabled.
- **Shared-PC Protection**: 
  - **15-Minute Inactivity Timeout**: Automatic logout for idle workstations.
  - **1-Hour Absolute Duration**: Force re-authentication every hour to ensure credentials remain valid.

### ⚡ Real-Time Operations
- **Instant WebSocket Updates**: Powered by **Socket.io**. Hardware events are pushed to the browser the millisecond they happen.
- **Live Door & Sensor Status**: Real-time visual badges for **Locked/Unlocked** (relay state) and **Open/Closed** (magnetic sensor) with zero-latency updates.
- **Smart Fallback**: Combined event-driven updates with a 5-second polling "heartbeat" for maximum reliability.

### 🏢 Site-Wide Management
- **🌐 Door Groups & Site Provisioning**: Group doors across multiple controllers and provision cards to an entire site in one click.
- **🔍 Searchable Multi-Select**: Integrated **Tom Select** for powerful, searchable multi-group selection.
- **💾 Persistent SQLite Backend**: All controller metadata, custom names, door groups, and network settings are saved permanently in a local database.

### 📊 Audit & Diagnostics
- **Unified Event History**: Automatically merges logs from all controllers into a single chronological audit trail.
- **Smart Classification**: Distinguishes between **Card Swipes** (Access) and **Sensor Events** (Hardware status) with hardware "Reason" code extraction.
- **System Debug**: Dedicated comms tester to send raw commands directly to controllers and inspect JSON responses.

## 🛠️ Installation & Setup

### 1. Prepare the Environment
The application is pre-configured to run from `/opt/door-controller`.

```bash
sudo mkdir -p /opt/door-controller
sudo chown $USER:$USER /opt/door-controller
cd /opt/door-controller
git clone https://github.com/your-repo/door-control-pro.git .
npm install
```

### 2. Configure OIDC Provider (Keycloak)
For the authentication to function correctly, your OIDC client must be configured with the following settings:

*   **Client ID**: Match your `.env` `OIDC_CLIENT_ID` (e.g., `door-control-pro`).
*   **Client Authentication**: `On` (Confidential Client).
*   **Authentication Flow**: Standard Flow (Authorization Code Flow) enabled. Implicit Flow disabled.
*   **Valid Redirect URIs**: `https://your-domain.com/callback` (Must be absolute).
*   **Valid Post Logout Redirect URIs**: `https://your-domain.com/`
*   **Web Origins**: `https://your-domain.com` (Required for CORS/WebSockets).
*   **Back-Channel Logout URL**: `https://your-domain.com/backchannel-logout` (If using the OIDC library's native handler).
*   **Advanced Settings**:
    *   **Response Mode**: `form_post` (This is required for large token payloads behind Nginx/Cloudflare).

### 3. Configure Environment Settings
| Variable | Description |
| :--- | :--- |
| `BASE_URL` | The external HTTPS URL of your application. |
| `OIDC_ISSUER_URL` | The discovery URL of your OIDC provider. |
| `OIDC_CLIENT_SECRET`| The secret key generated by your OIDC provider. |
| `UHPPOTE_BROADCAST` | The directed broadcast address for your subnet (e.g., `192.168.1.255:60000`). |

### 3. Setup Nginx
Nginx must handle large OIDC headers and WebSocket passthrough.

```nginx
location / {
    proxy_pass http://127.0.0.1:4000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    
    # REQUIRED: Buffers for OIDC Tokens
    proxy_buffer_size          128k;
    proxy_buffers              4 256k;
    proxy_busy_buffers_size    256k;
}
```
